# Учебный офис RAG

`ai-agent-hse` - это AI-ассистент учебного офиса, который ищет информацию по локальной базе документов и отвечает на вопросы через:

- HTTP API
- встроенный веб-дашборд
- административную панель
- Telegram-бота

Система индексирует документы из каталога `docs/`, сохраняет метаданные и текстовые чанки в PostgreSQL, а затем использует retrieval + LLM для генерации ответа.

## Что умеет проект

- индексировать `pdf`, `docx`, `dotx`, `xlsx`, `xls`, `csv`
- хранить документы как `docs + chunks` в PostgreSQL
- искать по содержимому, названию и описанию документа
- отвечать через `POST /ask`
- показывать пользовательский дашборд на `/`
- поддерживать административную панель на `/admin`
- запускать Telegram-бота с меню и частыми темами
- работать с OpenAI-compatible LLM
- запускаться одной командой через Docker Compose

## Архитектура в двух словах

В проекте нет отдельного frontend-сервиса. Статические файлы интерфейса находятся в `app/static/` и обслуживаются тем же FastAPI-приложением, которое отдает API.

Текущая схема запуска через Docker Compose:

- `postgres` - база данных
- `api` - FastAPI, пользовательский дашборд и админка
- `bot` - Telegram-бот

API и бот используют одну и ту же БД и один и тот же прикладной код. Telegram-бот не вызывает HTTP API, а работает с общими Python-модулями напрямую.

## Быстрый старт

### Вариант 1. Весь стек одной командой

1. Создайте `.env`:
   ```bash
   copy .env.example .env
   ```
2. При необходимости заполните:
   - `LLM_API_KEY`
   - `TELEGRAM_BOT_TOKEN`
   - `ADMIN_PASSWORD`
   - `ADMIN_SESSION_SECRET`
3. Поднимите весь стек:
   ```bash
   docker compose up --build
   ```

Что запустится:

- PostgreSQL
- API на `http://localhost:8000`
- пользовательский дашборд на `http://localhost:8000/`
- админка на `http://localhost:8000/admin`
- Telegram-бот

Python-зависимости из `requirements.txt` устанавливаются внутри Docker-образа на этапе сборки.

### Вариант 2. Локальный запуск без полного Docker-стека

1. Создайте `.env`:
   ```bash
   copy .env.example .env
   ```
2. Поднимите только Postgres:
   ```bash
   docker compose up -d postgres
   ```
3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
4. Запустите API:
   ```bash
   uvicorn app.main:app --reload
   ```
5. При необходимости отдельно запустите Telegram-бота:
   ```bash
   python -m bot.main
   ```

## Переменные окружения

Основные переменные:

- `DATABASE_URL` - строка подключения к PostgreSQL
- `DATA_DIR` - каталог с документами, по умолчанию `./docs`
- `LOG_LEVEL` - уровень логирования
- `AUTO_INDEX_ON_STARTUP` - автозапуск индексации на старте

LLM:

- `LLM_BASE_URL`
- `LLM_API_KEY`
- `LLM_MODEL`

Telegram:

- `TELEGRAM_BOT_TOKEN`
- `TELEGRAM_ALLOWED_CHAT_IDS`

Админка:

- `ADMIN_USERNAME`
- `ADMIN_PASSWORD`
- `ADMIN_SESSION_SECRET`

В Docker Compose переменные `DATABASE_URL` и `DATA_DIR` переопределяются автоматически под контейнерную схему.

## Индексация документов

Индексация может запускаться:

- автоматически при старте API
- автоматически при старте бота
- через CLI
- через HTTP API
- при загрузке документа через админку

CLI:

```bash
python -m scripts.index --path ./docs
```

Через API:

```bash
curl -X POST http://localhost:8000/index ^
  -H "Content-Type: application/json" ^
  -d "{\"path\":\"./docs\"}"
```

Особенности индексации:

- рассчитывается checksum файла
- неизменившиеся документы не переиндексируются
- для документов формируется описание
- текст режется на чанки
- поиск использует PostgreSQL Full-Text Search

## API

Основные эндпоинты:

- `GET /health`
- `POST /ask`
- `POST /index`
- `GET /api/docs`
- `GET /api/docs/{doc_id}/file`
- `GET /api/stats`
- `GET /api/history`

Проверка сервиса:

```bash
curl http://localhost:8000/health
```

Пример запроса к агенту:

```bash
curl -X POST http://localhost:8000/ask ^
  -H "Content-Type: application/json" ^
  -d "{\"question\":\"Когда проходит сессия 3 модуля?\",\"top_k\":5}"
```

Пример ответа:

```json
{
  "answer": "Найденные источники и релевантные фрагменты: ...",
  "sources": [
    {
      "title": "СЕССИЯ 3 модуль (c 25.03.2026 по 28.03.2026) с изм. 20.03.26",
      "file_path": "/app/docs/...",
      "chunk_index": 0,
      "snippet": "..."
    }
  ]
}
```

## Пользовательский дашборд

После запуска API откройте:

```text
http://localhost:8000/
```

Что есть на странице:

- форма вопроса к агенту
- список найденных источников
- статистика по документам и чанкам
- история запросов текущего процесса
- кнопка переиндексации `docs`

История запросов хранится в памяти процесса и очищается после перезапуска.

## Административная панель

Админка доступна по адресу:

```text
http://localhost:8000/admin
```

Что умеет:

- вход администратора
- просмотр списка документов
- загрузка нового документа
- редактирование названия и описания
- удаление документа
- открытие или скачивание файла

Админская авторизация включается только если заданы:

- `ADMIN_USERNAME`
- `ADMIN_PASSWORD`
- `ADMIN_SESSION_SECRET`

Если эти переменные не заполнены, CRUD-операции будут недоступны.

## Telegram-бот

Для работы бота нужен токен:

```env
TELEGRAM_BOT_TOKEN=your_token
```

Опционально можно ограничить доступ конкретными чатами:

```env
TELEGRAM_ALLOWED_CHAT_IDS=123456789,987654321
```

Запуск отдельно:

```bash
python -m bot.main
```

Или вместе со всем стеком:

```bash
docker compose up --build
```

Что умеет бот:

- `/start`
- `/menu`
- кнопка `Задать вопрос`
- кнопка `Документы`
- кнопка `Частые темы`
- кнопка `Как пользоваться`
- показ источников отдельной кнопкой
- постраничный просмотр списка документов

Частые темы включают:

- перевод
- сессию
- ВКР
- расписание
- физкультуру
- устав

Бот использует тот же RAG-пайплайн, что и API.

## LLM

Если LLM не настроена:

- система все равно работает
- ответ строится как fallback по найденным материалам

Если LLM настроена, используются:

```env
LLM_BASE_URL=https://api.openai.com/v1
LLM_API_KEY=...
LLM_MODEL=gpt-4o-mini
```

LLM применяется:

- для генерации ответа пользователю
- для генерации кратких описаний документов при индексации

## Структура проекта

- `app/main.py` - FastAPI, API, дашборд, админка
- `app/indexer.py` - индексация документов
- `app/search.py` - FTS и fallback-поиск
- `app/rag.py` - общий пайплайн ответа
- `app/llm.py` - клиент OpenAI-compatible LLM
- `app/models.py` - модели БД
- `app/admin_auth.py` - админская аутентификация
- `app/static/` - пользовательский и административный интерфейс
- `bot/main.py` - Telegram-бот
- `scripts/index.py` - CLI-индексация
- `docker-compose.yml` - запуск полного стека
- `Dockerfile` - образ для API и бота

## Ограничения текущей версии

- история запросов хранится в памяти процесса
- поиск основан на PostgreSQL FTS и не использует embeddings
- качество ответа зависит от качества извлечения текста из документов
- Telegram-бот и API используют общую БД, но не разделяют in-memory историю
- LLM является внешней сетевой зависимостью
